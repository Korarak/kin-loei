**โปรไฟล์:** แพ้ถั่วลิสง, ถั่วเปลือกแข็ง, งา

| additives | E320 (สารกันบูด BHA) |
| label_allergen_warnings | "ผลิตในโรงงานที่ใช้ถั่วเปลือกแข็งและงา" |

## Test Case 4 — ถั่วฝักยาวแช่แข็ง CP (โรคเก๊าท์ + ไตวายเรื้อรัง)

**สินค้า:** ถั่วฝักยาวแช่แข็ง — เจริญโภคภัณฑ์ (CP)  
**โปรไฟล์:** โรคเก๊าท์, ไตวายเรื้อรัง, เลี่ยงโพแทสเซียมสูงและพิวรีนสูง

**ผลที่ได้**

| Field | ค่า |
|-------|-----|
| Status | ⚠️ **CAUTION** (ไม่ใช่ AVOID) |
| flagged | ถั่วฝักยาว — **medium** |
| additives | ไม่มี |
| allergen warnings | ไม่มี |

**Summary:**  
> "เป็นผักธรรมชาติ 100% แต่มีพิวรีนและโพแทสเซียมซึ่งเป็นสารที่ผู้ป่วยโรคเก๊าท์และไตวายเรื้อรังต้องระมัดระวัง"

**Recommendation:**  
> "ควรจำกัดปริมาณในแต่ละมื้อ และแนะนำให้**ลวกในน้ำเดือดแล้วเทน้ำทิ้ง**ก่อนนำไปปรุงอาหาร เพื่อลดปริมาณโพแทสเซียม"

**สิ่งที่โดดเด่นในเคสนี้:**
- Gemini ให้ CAUTION ไม่ใช่ AVOID — ถูกต้องเพราะถั่วฝักยาวเป็นผักธรรมชาติไม่ได้อันตรายโดยตรง แค่ต้องระวัง
- severity = **medium** ไม่ใช่ high — สะท้อนความเสี่ยงได้สมเหตุสมผล
- คำแนะนำ "ลวกแล้วเทน้ำทิ้ง" เป็นเทคนิคทางโภชนาการที่ใช้จริงในทางคลินิกสำหรับผู้ป่วยไต

**การประเมิน:** ✅ ดีมาก — แสดงให้เห็นว่าระบบแยกแยะ "อันตราย" กับ "ต้องระวัง" ได้ และให้คำแนะนำเชิงปฏิบัติที่มีคุณค่าทางการแพทย์

## สิ่งที่น่าสนใจที่พบ

### 1. แยก CAUTION / AVOID ได้อย่างมีเหตุผล
ถั่วฝักยาวได้ CAUTION (ไม่ใช่ AVOID) เพราะเป็นผักธรรมชาติที่ไม่อันตรายในตัวเอง แต่ต้องระวังปริมาณ ระบบไม่ได้ over-flag ทุกอย่างเป็น AVOID

### 2. ตรวจ Cross-Contamination ได้
ถั่วลิสงคาราบาว — อ่านข้อความ "ผลิตในโรงงานที่ใช้ถั่วเปลือกแข็งและงา" จากฉลากแล้ว flag เป็น high severity ทั้งที่ไม่ใช่ส่วนประกอบโดยตรง — พฤติกรรมนี้ถูกต้องและสำคัญมากสำหรับผู้แพ้รุนแรง

### 3. คำแนะนำเชิงปฏิบัติ
ไม่ได้บอกแค่ "ห้ามกิน" แต่ให้ทางเลือกด้วย เช่น:
- น้ำตาล → สารให้ความหวานทดแทน
- แป้งสาลี → แป้งข้าวเจ้า / แป้งอัลมอนด์
- ถั่วฝักยาว → ลวกน้ำเดือดแล้วเทน้ำทิ้งเพื่อลด K⁺

### 4. วิตามินเสริมไม่ถูก Overflag
แป้งสาลีมี B1, B2, ไนอาซิน, ธาตุเหล็กเสริม — Gemini รู้ว่าสิ่งเหล่านี้เป็นวิตามินปกติ ไม่ใช่ additive อันตราย ไม่ flag โดยไม่จำเป็น

## ปัญหาที่พบ — Google Search Grounding

ฟีเจอร์ค้นหาข้อมูลเปรียบเทียบ (Pass 2) ยังไม่ทำงานในทุก test case

**Error Log:**
```
[Gemini Search] grounding gemini-2.0-flash:
  → 429 RESOURCE_EXHAUSTED (quota หมด หรือ key ไม่รองรับ grounding)

[Gemini Search] grounding gemini-2.5-flash-preview:
  → 404 NOT_FOUND (model ไม่รองรับ API version v1beta)

[Gemini Search] grounding gemini-2.5-flash-preview-05-20:
  → 404 NOT_FOUND (model ไม่รองรับ API version v1beta)

[DDG Fallback]:
  → Response ว่างเปล่า (สินค้าไทยไม่มีข้อมูลใน DuckDuckGo Instant Answer)
```

**สาเหตุ:**

| Error | สาเหตุ |
|-------|--------|
| 429 `gemini-2.0-flash` | API key ปัจจุบันไม่มี Google Search Grounding quota |
| 404 model อื่น | Model name ไม่ตรงกับ API version ที่ key เข้าถึงได้ |
| DDG ว่าง | Instant Answer API ไม่มีข้อมูลสินค้าไทย |

**แนวทางแก้ไข:**

1. เปิด Google Search Grounding ใน [Google AI Studio](https://aistudio.google.com) → Billing / API settings
2. เพิ่ม **Tavily Search API** (`TAVILY_API_KEY`) ใน `.env` — ออกแบบมาสำหรับ AI โดยเฉพาะ, free 1,000 req/เดือน, รองรับภาษาไทย
3. หรือใช้ **SerpAPI** / **Brave Search API** สำหรับผลลัพธ์เว็บจริง
